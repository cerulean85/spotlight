## 요약

AI/ML 기반의 레시피 자동 추천 시스템 개발에 참여하여, 기존 수동 튜닝 방식을 데이터 기반 공정으로 전환하는 데 기여하였습니다. 데이터 파이프라인 구축 및 모델 고도화를 통해 레시피 설계 시간을 50% 단축시켰으며, 숙련도에 상관없는 공정 표준화를 실현했습니다.

## 프로젝트 배경

디스플레이 제조 공정에서 <strong>퍼니스(Furnace, 가열로)</strong>는 패널의 품질과 신뢰성을 좌우하는 핵심 열처리 장비입니다. 이는 단순히 열을 가하는 것을 넘어, 소재의 화학적 반응을 유도하고 물리적 성질을 결정짓는 중요한 역할을 수행합니다.

디스플레이가 대형화·고해상도화될수록 <strong>온도 균일도(Uniformity)</strong>는 생명과도 같습니다. 퍼니스 내부 온도가 단 1도만 차이가 나도 화면 밝기가 불균일해지는 화면 얼룩(Mura) 현상이 발생할 수 있기 때문입니다. 특히 픽셀 스위치 역할을 하는 <strong>TFT(박막 트랜지스터)</strong>는 열처리에 매우 민감하여, 미세한 온도 편차에도 결정화 정도나 산화물 반도체의 결합 구조가 달라집니다. 이로 인해 위치마다 <strong>문턱 전압(Threshold Voltage)</strong> 값이 달라지면 동일한 신호에도 밝기가 제각각인 얼룩이 발생하게 됩니다.

기판이 커질수록 중앙부와 가장자리의 온도 차를 줄이는 것은 기하급수적으로 어려워지며, 이를 제어하기 위해 퍼니스 내부를 수십 개의 독립적인 <strong>존(Zone)</strong>으로 나누어 히터와 센서를 개별 제어하고, 고온 기체를 순환시키는 등 고도화된 기술이 적용되고 있습니다.

## 프로젝트 개요

<strong>한화모멘텀(주)</strong>에서 2024년 수행한 **멀티 스텝 퍼니스 레시피 자동 추천 시스템** 개발 프로젝트는 이러한 열 균일도 제어를 자동화하기 위해 시작되었습니다.

기존의 다단계(Multi-step) 가열 공정 레시피는 숙련된 엔지니어의 감과 경험에 의존해 수동으로 튜닝되었습니다. 이로 인해 작업자에 따라 품질 편차가 발생하고, 최적화에 많은 시간이 소요되는 문제가 있었습니다. 본 프로젝트는 AI/ML 기술을 도입하여 레시피 도출 프로세스를 정량화하고, 누가 작업하더라도 최상의 품질을 보장하는 것을 목표로 했습니다.

제공해주신 시스템 아키텍처 다이어그램과 작성하신 초안을 바탕으로, **데이터의 흐름(Flow)**과 **각 컴포넌트의 역할**이 명확히 드러나도록 퇴고해 보았습니다.

특히 다이어그램에 포함되어 있지만 원문에 누락된 **React Client(사용자 인터페이스)**에 대한 설명을 추가하고, Database Tier의 성격(NoSQL)을 구체화하여 전문성을 높였습니다.


## 시스템 구성

<div className="flex justify-center">
  <Image src={systemArchitecture} alt="System Architecture Diagram" style={{ width: '66.67%', height: 'auto' }} />
</div>

이 시스템은 크게 **ECS, Application, Database**의 3개 Tier로 구성되며, 퍼니스 공정의 정밀한 제어와 데이터 분석을 위해 유기적으로 연결되어 있습니다.

**1) ECS Tier (현장 제어 영역)**: HMI, PLC, MSSQL로 구성되며, 퍼니스 장비의 센서에서 발생하는 모든 온도 및 공정 데이터를 수집합니다. 수집된 데이터는 PLC를 통해 상위 단계인 Application Tier로 실시간 제공됩니다.

**2) Application Tier (데이터 처리 및 중계 영역)**: 이 영역은 데이터의 수집, 가공, 그리고 AI 분석을 위한 중계 역할을 수행합니다.

* **Machine Agent:** PLC와 직접 통신하여 센서의 실시간 온도 데이터를 읽어 들이고, 이를 **Node.js Server**로 전달하는 가교 역할을 합니다. (주로 소켓 통신 등을 활용)
* **Node.js Server (Backend):** 시스템의 허브(Hub)로서, 수신된 데이터를 **React Client**로 전송하여 작업자가 모니터링할 수 있게 하고, 동시에 분석이 필요한 데이터는 **RabbitMQ**를 통해 AI Server로 전송합니다.
* **AI Server:** RabbitMQ(MQTT 프로토콜 등)를 통해 전달받은 온도 데이터를 바탕으로 공정 이상 유무를 판단하거나 예지 보전 모델을 수행합니다.

**3) Database Tier (데이터 저장 영역)**: **MongoDB**를 기반으로 구축된 이 계층은 정형 데이터뿐만 아니라, Node.js Server로부터 전달받은 방대한 양의 온도 데이터와 설비 로그를 **비정형(NoSQL)** 포맷으로 유연하게 저장 및 관리합니다.

* **피드백 루프**: 실제 적용된 레시피의 품질 데이터를 다시 모델에 학습시키는 지속적 개선(CI) 구조 확립

## Abstract

Contributed to the development of an AI/ML system that automates furnace recipe recommendations, successfully transitioning from manual tuning to a data-driven approach. This contribution helped reduce recipe design time by 50% and ensured consistent thermal uniformity across the process.

## Project Background

In the display manufacturing process, the **Furnace** is a critical thermal treatment system that dictates panel quality and reliability. Beyond simply applying heat, it plays a vital role in inducing chemical reactions and determining the physical properties of materials.

As displays become larger and reach higher resolutions, **Temperature Uniformity** becomes paramount. A discrepancy of even  inside the furnace can cause "Mura" (display blemishes), where screen brightness becomes uneven. Specifically, **TFTs (Thin-Film Transistors)**, which act as pixel switches, are extremely sensitive to thermal treatment. Even minor temperature deviations can alter the degree of crystallization or the bonding structure of oxide semiconductors. This leads to variations in Threshold Voltage () across the panel, resulting in inconsistent brightness for the same input signal.

As substrate sizes increase, reducing the temperature gap between the center and the edges becomes exponentially difficult. To control this, advanced technologies are employed, such as dividing the furnace into dozens of independent zones for individual heater/sensor control and implementing high-temperature gas circulation.

## Project Overview

The "Multi-Step Furnace Recipe Auto-Recommendation System" development project, conducted at Hanwha Momentum in 2024, was initiated to automate this thermal uniformity control.

Previously, multi-step heating process recipes were manually tuned based on the intuition and experience of skilled engineers. This led to quality deviations depending on the operator and required significant time for optimization. This project aimed to quantify the recipe derivation process by introducing AI/ML technology, ensuring peak quality regardless of the operator's experience level.

## System Architecture

<div className="flex justify-center">
  <Image src={systemArchitecture} alt="System Architecture Diagram" style={{ width: '66.67%', height: 'auto' }} />
</div>

This system comprises three main tiers: **ECS, Application, and Database**. These tiers are seamlessly interconnected to ensure precise control of the furnace process and enable effective data analysis.

**1) ECS Tier (Field Control Layer)**: Composed of HMI, PLC, and MSSQL, this tier collects all temperature and process data generated by the furnace equipment sensors. The collected data is transmitted in real-time to the upper-level Application Tier via the PLC.

**2) Application Tier (Data Processing & Relay Layer)**: This tier serves as an intermediary for data collection, processing, and AI analysis.

* **Machine Agent:** Acts as a bridge by communicating directly with the PLC to retrieve real-time temperature data from sensors and transmitting it to the **Node.js Server** (typically utilizing socket communication).
* **Node.js Server (Backend):** Functioning as the system hub, it sends received data to the **React Client**, enabling operators to monitor the process. Simultaneously, it transmits data requiring analysis to the **AI Server** via **RabbitMQ**.
* **AI Server:** Based on the temperature data received via RabbitMQ (using protocols like MQTT), this server detects process anomalies or executes predictive maintenance models.

**3) Database Tier (Data Storage Layer)**: Built upon **MongoDB**, this tier flexibly stores and manages not only structured data but also vast amounts of temperature data and equipment logs received from the Node.js Server in an **unstructured (NoSQL)** format.

## Key Roles & Achievements

I contributed to the entire lifecycle of the project, from building the data pipeline to modeling and system implementation.

### 1) Data Collection & Preprocessing Pipeline Design

* **Collection**: Real-time acquisition of core process data, including temperature, time, and heating rates, from thermal equipment sensors.
* **Processing**: Enhanced data quality through noise reduction, missing value imputation, and data standardization.
* **Storage**: Established a database using **MongoDB** to create an efficient environment for analyzing and training large-scale log data.

### 2) AI/ML-Based Recipe Recommendation Engine Development

* **Modeling**: Developed regression and classification models to predict optimal temperatures for specific conditions by training on historical thermal treatment data.
* **Validation**: Ensured model reliability through cross-validation between simulation data and actual production floor data.

### 3) System Implementation & Optimization

* **UI/UX**: Designed an intuitive interface allowing field engineers to input conditions and verify recommended recipes easily.
* **Feedback Loop**: Established a **Continuous Improvement (CI)** structure where quality data from applied recipes is fed back into the model for retraining.

## Final Results

By implementing this system, we **reduced recipe design and testing time by more than 50%**. Furthermore, it significantly increased operational flexibility by enabling any operator to utilize high-quality recipes regardless of their skill level and established a foundation for rapid response to new processing conditions.
